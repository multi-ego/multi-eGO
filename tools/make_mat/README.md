# Interatomic contact matrices tools

## make_mat.py

Calculates intra- and inter-molecular contact probability matrices from MD histogram data.
For each atom pair it computes a c12-weighted average distance and a contact probability,
writing the results as compressed HDF5 files (`intramat_*.ndx.h5` / `intermat_*.ndx.h5`).

### Usage

```bash
python make_mat.py \
  --target_top <target_topology> \
  --mego_top   <mego_topology>   \
  [--histo <histogram_dir_or_tar>] \
  [--mode  intra+same+cross]     \
  [options]
```

### Arguments

| Argument | Required | Default | Description |
|---|---|---|---|
| `--target_top` | yes | — | Path to the all-atom topology on which histograms were calculated |
| `--mego_top` | yes | — | Path to the multi-eGO topology generated by `gmx pdb2gmx` |
| `--histo` | yes* | — | Path to the histogram directory (plain or `.tar`) or HDF5 store. Required unless `--zero` is set |
| `--zero` | no | `False` | Write zero-probability matrices (correct cutoffs, zero distances and probabilities). Mutually exclusive with `--histo` |
| `--mode` | no | `intra+same+cross` | Which matrices to compute. Any `+`-separated combination of `intra` (intra-molecular), `same` (inter-molecular, same species), `cross` (inter-molecular, different species) |
| `--out` | no | `./` | Output directory (must exist) |
| `--out_name` | no | — | Optional string inserted into output filenames: `intramat_<out_name>_mi_mj.ndx.h5` |
| `--num_threads` | no | `1` | Number of parallel worker processes for histogram analysis |
| `--cutoff` | no | `0.75` | Maximum cutoff (nm) used when accumulating histograms |
| `--bkbn_H` | no | `H` | Name of the backbone hydrogen atom (use `HN` for CHARMM topologies) |
| `--nb` | no | `False` | Retain only contacts between neighbouring residues (|Δres| ≤ 2) |
| `--nonb` | no | `False` | Retain only contacts between non-neighbouring residues (|Δres| > 2) |
| `--tar` | no | `False` | Read histograms from a `.tar` archive instead of a directory |
| `--noh5` | no | `False` | Read histograms from plain text (`.dat`) files instead of HDF5 |
| `--custom_c12` | no | — | Path to a file with custom c12 parameters for special molecule types |

### Output

One HDF5 file per molecule-pair combination, named:

```
<out>/intramat_[<out_name>_]<mi>_<mj>.ndx.h5
<out>/intermat_[<out_name>_]<mi>_<mj>.ndx.h5
```

Each file contains a table with columns:
`molecule_name_ai`, `ai`, `molecule_name_aj`, `aj`, `distance`, `probability`, `cutoff`, `learned`.

### Examples

Compute all matrices (intra + same-species inter + cross-species inter) from HDF5 histograms:

```bash
python make_mat.py \
  --histo    md_run/histograms \
  --target_top system.top \
  --mego_top   mego.top
```

Compute only the intramolecular matrix using 4 parallel threads:

```bash
python make_mat.py \
  --histo      md_run/histograms \
  --target_top system.top \
  --mego_top   mego.top \
  --mode       intra \
  --num_threads 4
```

Generate a zero-probability matrix (useful as a starting template):

```bash
python make_mat.py \
  --zero \
  --target_top system.top \
  --mego_top   mego.top \
  --out        zero_matrices/
```

## ndx2HDF5.py

Converts a plain-text (or `.gz`) contact matrix to a compressed HDF5 file.
Useful for large systems where HDF5 read performance is significantly faster than
re-parsing the text file on every `make_mat` run.

### Usage

```bash
python ndx2HDF5.py -i <input_file> [-o <output_file>]
```

### Arguments

| Argument | Required | Default | Description |
|---|---|---|---|
| `-i` / `--input` | yes | — | Path to the plain-text matrix (`.ndx`, `.ndx.gz`, or similar) |
| `-o` / `--output` | no | auto | Output HDF5 path. Defaults to the input path with `.gz` stripped and `.h5` appended, or `.h5` appended directly for other extensions |

### Examples

```bash
# Convert a plain .ndx file
python ndx2HDF5.py -i intramat_1_1.ndx

# Convert a gzip-compressed file to a specific output path
python ndx2HDF5.py -i intramat_1_1.ndx.gz -o matrices/intramat_1_1.ndx.h5
```

## HDF52ndx.py

Converts a compressed HDF5 contact matrix back to plain-text format.
Useful for inspecting matrix contents or passing them to downstream tools that
do not read HDF5.

### Usage

```bash
python HDF52ndx.py -i <input_file> [-o <output_file>]
```

### Arguments

| Argument | Required | Default | Description |
|---|---|---|---|
| `-i` / `--input` | yes | — | Path to the HDF5 contact matrix file (`.ndx.h5`) |
| `-o` / `--output` | no | auto | Output text path. Defaults to the input path with the `.h5` extension removed |

### Examples

```bash
# Restore the default plain-text name (intramat_1_1.ndx)
python HDF52ndx.py -i intramat_1_1.ndx.h5

# Write to a specific output path
python HDF52ndx.py -i intramat_1_1.ndx.h5 -o output/intramat_1_1.ndx
```
